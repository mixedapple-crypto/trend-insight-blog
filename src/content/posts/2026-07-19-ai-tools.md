## 1. Claude Code 자동진행 기능: 무음 60초 후 자동 실행 리스크

**Claude Code 2.1.198**이 자동진행(auto-advance) 기능을 탑재했습니다. `AskUserQuestion` 프롬프트에 60초간 응답이 없으면 에이전트(Agent)가 독자적으로 판단해 다음 작업을 실행합니다.

문제는 이 기능이 배포 시 별도 문서 없이 기본 활성화 상태였다는 점입니다. 파일 수정, 외부 도구 호출 같은 비가역적 작업도 사용자 확인 없이 진행될 수 있습니다.

**지금 바로 해야 할 조치**는 다음과 같습니다.

- Claude Code 설정(Settings)에서 auto-advance 항목을 찾아 비활성화합니다.
- 장시간 자리를 비우는 작업 세션에서는 특히 확인이 필요합니다.
- CI/CD 파이프라인처럼 무인 실행 환경에서는 설정 상태를 명시적으로 검증하십시오.

강의 환경에서 Claude Code를 시연할 때, 수강생이 자리를 비운 사이 실습 코드가 자동 변경되는 상황을 방지하려면 이 설정을 미리 꺼두는 것이 좋습니다. [출처](https://news.hada.io/topic?id=31549)

## 2. SQLite Query Explainer: 브라우저 기반 쿼리 계획 해설기

Simon Willison이 **SQLite Query Explainer**를 공개했습니다. 브라우저 안에서 Pyodide(WebAssembly 기반 Python 런타임)로 SQLite를 실행하고, `EXPLAIN` 및 `EXPLAIN QUERY PLAN` 결과를 Claude가 자연어로 해설해 줍니다.

설치가 필요 없습니다. SQL 쿼리를 붙여 넣으면 원시 쿼리 계획과 함께 "데이터베이스 엔진이 어떤 순서로 어떤 인덱스를 쓰는지"를 평문으로 확인할 수 있습니다.

실전 활용 포인트는 다음과 같습니다.

- 인덱스 없이 전체 테이블을 스캔하는 쿼리를 시각적으로 파악할 때 유용합니다.
- 복잡한 JOIN 순서를 최적화하기 전 계획을 비교해 보는 용도로 씁니다.
- Willison은 Claude 설명이 검증되지 않았으므로 성능 판단의 보조 자료로만 쓰라고 명시했습니다.

SQL 최적화를 가르치는 강의에서 "EXPLAIN 결과를 어떻게 읽나요?"라는 질문을 받을 때, 이 도구를 라이브 시연에 바로 활용할 수 있습니다. [출처](https://simonwillison.net/2026/Jul/18/sqlite-query-explainer)

## 3. Fable 5 구독 영구화: Max·Team Premium 편입 변경

Anthropic이 **Fable 5** 모델을 Max 및 Team Premium 구독에 영구 포함시켰습니다. 애초 계획은 API 전용 접근이었지만 방향을 바꿨습니다.

구독별 변경 내용을 정리하면 다음과 같습니다.

| 구독 | 변경 내용 |
|------|-----------|
| Max · Team Premium | Fable 5 영구 포함, 사용량 50% 한도 |
| Pro · Team Standard | 기존 크레딧 유지 + 1회성 100달러 크레딧 추가 |
| 20달러/월 플랜 | Fable 5 미포함 |

배경에는 **GPT-5.6 Sol**과의 경쟁 압박이 있습니다. 최상위 모델을 구독 단계에서 잠그면 이탈 유인이 생기기 때문에 Anthropic이 접근 정책을 수정했습니다.

GPU 자원이 학습에서 추론 쪽으로 재배분될 가능성이 있으므로, 응답 속도 변화를 실제 워크플로에서 모니터링하는 것이 좋습니다. [출처](https://simonwillison.net/2026/Jul/18/claude-make-fable-5-permanent)

## 4. Kimi K3 vs Claude: 가격 구조 차이와 실전 비교

중국 Moonshot AI의 **Kimi K3**가 API 대안으로 주목받고 있습니다. 가격은 입력 토큰 100만 개당 3달러, 출력 토큰 100만 개당 15달러입니다. Claude 3.5 기준 입력 10달러, 출력 50달러와 비교하면 약 70% 낮습니다.

실사용 보고에서 일상적인 코딩 작업 기준으로 결과물 품질과 토큰 효율 차이를 구별하기 어렵다는 평가가 나왔습니다. 고가의 모델이 모든 작업에서 우위를 보이는 것은 아니라는 점을 이 사례가 다시 확인해 줍니다.

실전 판단 기준을 정리하면 다음과 같습니다.

- 반복 배치 작업, 코드 리뷰 자동화처럼 **토큰 소비량이 많은 워크플로**에서 비용 효과가 큽니다.
- 맥락 유지가 중요한 장문 대화나 복잡한 멀티 에이전트 파이프라인은 별도 평가가 필요합니다.
- 국내 서비스 가용성과 데이터 거주(data residency) 정책을 먼저 확인하십시오.

오늘 신호를 하나로 묶으면, AI 도구 생태계는 "설정 기본값의 리스크 관리"와 "비용 최적화" 두 축에서 동시에 움직이고 있습니다. Claude Code의 자동진행 기본 활성화는 편의와 통제 사이의 균형이 설계 단계에서 얼마나 중요한지 보여주는 사례이고, Kimi K3의 부상은 최고 성능 모델 외에도 실용적 대안을 검토하는 것이 이제 선택이 아닌 기본 루틴임을 보여줍니다. [출처](https://news.hada.io/topic?id=31574)
