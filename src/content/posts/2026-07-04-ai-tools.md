## 1. Claude Code 모델 위임: 하위 에이전트로 비용 절감하기

Fable(Fable) 가격 인상 이전, Simon Willison이 공유한 실전 프롬프트가 주목받고 있습니다. [출처](https://simonwillison.net/2026/Jul/3/judgement)

핵심 명령어는 한 줄입니다.

> "For all coding tasks use your judgement to decide an appropriate lower power model and run that in a subagent"

이 문장을 Claude Code에 입력하면, 도구가 자동으로 이 지시를 **메모리 파일**로 저장합니다. 이후 세션에서도 자동 적용됩니다.

위임 구조는 다음과 같이 작동합니다.

- 실질적 코딩 구현 → **Sonnet** 하위 에이전트가 담당
- 주석 추가·간단 수정 등 기계적 편집 → **Haiku** 하위 에이전트 배정
- 설계 검토·판단 종합 → 메인 루프(Fable/Opus)가 처리

실전 팁은 두 가지입니다.

첫째, 명령어 입력 후 즉시 메모리 파일 저장 경로를 확인하십시오. 팀 공유 리포지토리에 커밋하면 동료 모두가 동일 비용 절감 구조를 사용할 수 있습니다.

둘째, 복잡한 리팩터링은 Sonnet에게, 단순 변수명 교체는 Haiku에게 명시적으로 분리 지시하면 모델이 잘못된 위임을 방지합니다.

강의 환경에서는 수강생이 토큰 예산을 제어하는 방법을 직접 설명하는 예제로 활용할 수 있습니다.

## 2. 오픈소스 AI 갭 맵: 421개 제품 데이터를 직접 쿼리하는 법

비영리 단체 Current AI가 2026년 7월 3일 **Gap Map v0.1**을 공개했습니다. [출처](https://simonwillison.net/2026/Jul/3/open-source-ai-gap-map)

수록 규모는 아래와 같습니다.

- 심층 분석 제품 421개(도구 266, 모델 85, 데이터셋 50, 하드웨어 20)
- 14개 카테고리 × 3개 스택 레이어 분류
- 미분류 아티팩트(artifact) 24,400개
- GitHub 추적 리포지토리 16,185개 CSV 공개

데이터는 **MIT 라이선스** YAML 파일 1,184개로 구성됩니다. `currentai-org/os-ai-map` 리포지토리에서 바로 접근할 수 있습니다.

가장 빠른 탐색 방법은 **Datasette Lite**입니다.

- CSV를 Datasette Lite에 드래그하여 로드
- `category`, `organization`, `stack_layer` 컬럼으로 필터
- 특정 카테고리의 오픈소스 대안을 즉시 발견 가능

실전 팁: 현재 사용 중인 독점 도구명을 `organization` 컬럼에서 검색하면, 동일 카테고리의 오픈소스 대체재 목록이 바로 나옵니다.

강의에서 ML 기술 스택을 소개할 때 이 데이터셋을 기준 지도로 활용하면 최신 생태계 현황을 별도 조사 없이 제시할 수 있습니다.

## 3. Anthropic 3종 공개: Sonnet 5·Fable 5·Claude Science 실전 활용

Anthropic이 2026년 7월 초 세 가지 제품을 동시에 시장에 냈습니다. [출처](https://yozm.wishket.com/magazine/detail/3830)

각 제품의 실무 포지션은 구분됩니다.

**Sonnet 5**: GPT-4o 동급 성능을 절반 비용으로 제공합니다. 비용 민감 프로덕션 워크로드(workload)에 적합합니다.

**Fable 5**: 18일 중단 후 7월 1일 재개된 최상위 모델입니다. OpenAI의 GPT-5.6 지연 시기에 맞물려 출시된 점이 구조적 대비를 만들어 냅니다.

**Claude Science**: 과학·제약 분야 특화 연구 워크벤치(workbench)입니다. 일반 개발 도구보다 연구 기관·바이오테크 팀에 해당합니다.

실전 선택 기준은 단순합니다.

- API 호출 비용이 핵심 제약이라면 → Sonnet 5
- 설계·판단이 필요한 고복잡도 작업이라면 → Fable 5 메인 루프(1번 섹션의 위임 구조 적용)
- 논문 분석·실험 설계 자동화라면 → Claude Science 검토

한 제공사 안에서 비용-성능 티어를 구성할 수 있다는 점이 이번 3종 동시 공개의 실질적 의미입니다.

## 4. AI 에이전트 자율성 설계: '짧은 목줄'과 루프 에이전트 대비

에이전트 코딩의 실무 과제가 '프롬프트 작성'에서 '자율성 수준 설계'로 이동하는 신호가 동시에 포착됩니다. [출처](https://news.hada.io/topic?id=31115) [출처2](https://news.hada.io/topic?id=31113)

두 접근법은 상반된 전제에서 출발합니다.

**짧은 목줄(Short Leash)** 방식은 보안 민감 프로젝트에 적합합니다.

- AI가 생성한 변경 사항을 개발자가 단계별로 검토·승인
- 병렬 멀티 에이전트 '바이브 코딩'이 초래하는 코드베이스 이해 저하를 방지
- 장점: 의도 이탈 최소화, 단점: 속도 저하

**루프 에이전트(Retry-now)** 방식은 반복 리팩터링 자동화에 적합합니다. [출처](https://news.hada.io/topic?id=31111)

- 매 반복마다 **컨텍스트 윈도우를 초기화**하여 재분석
- 이전 결정을 방어하려는 에이전트의 경향성을 제거
- 코드 품질 개선 자동화에 유효

자율성 수준 설계의 실전 원칙은 하나입니다. 작업 위험도(읽기 전용 쿼리 vs 프로덕션 배포)에 따라 권한과 검증 방식을 명시적으로 분리하십시오. 단일 자율성 수준을 모든 에이전트에 일괄 적용하면 복잡한 멀티 에이전트 워크플로에서 구조가 무너집니다.

## 5. 과장 vs 실속 — 교육자와 실무자가 마주한 두 가지 구조 변화

오늘 흐름에는 도구 기능과 함께 두 가지 구조 변화 신호가 섞여 있습니다.

첫 번째는 교육 시장입니다. Josh W. Comeau를 포함한 다수 강의 제작자가 전년 대비 **50% 이상** 판매 감소를 보고합니다. [출처](https://simonwillison.net/2026/Jul/3/josh-w-comeau)

원인은 두 가지입니다.

- 개발자 직업 불확실성 → 기술 투자 위축
- LLM 개인 튜터링이 유료 강의를 대체

이 흐름에서 강사에게 실용적인 선택지는 하나입니다. AI 도구 자체를 커리큘럼 중심에 놓고, 'AI가 대체하지 못하는 판단 훈련'을 학습 목표로 재설계하는 방향입니다.

두 번째는 15세 개발자 사례입니다. [출처](https://yozm.wishket.com/magazine/detail/3826)

코딩 경험 없이 AI만으로 협업 도구 'Wayver'를 제작하고 Reddit에 공개했으나, 반응은 제품-시장 적합성(PMF)의 부재를 드러냈습니다. AI가 개발 속도를 높이지만, 실사용자 검증은 대체하지 못한다는 점이 다시 확인된 사례입니다.

오늘의 AI 도구 흐름을 한 줄로 정리하면: 모델 성능 경쟁은 위임·자율성·비용 설계라는 실무 문제로 이동했고, 도구가 강력해질수록 설계 판단의 책임은 사람에게 집중됩니다.
