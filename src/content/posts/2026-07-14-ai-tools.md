## 1. Claude Code·GPT-5.6 사용 한도 해제: 지금 바로 써야 하는 이유

**OpenAI**는 GPT-5.6 Sol 수요 급증에 대응해 ChatGPT Plus·Pro·Business 구독자의 5시간 사용 한도를 일시 해제하고 쿼터를 전면 초기화했습니다.

GPT-5.6 Sol은 에이전트(Agent) 코딩 벤치마크 TerminalBench 2.1에서 91.9%를 기록했습니다. Claude의 83.4%와 비교하면 유의미한 격차입니다.

**Anthropic**도 맞불을 놨습니다. Claude Fable 5 무료 체험을 7월 19일까지 연장했고, Claude Code 주간 사용 한도를 같은 기간 50% 상향했습니다.

API 요금은 GPT-5.6 Sol 기준 입력 100만 토큰당 5달러, 출력 100만 토큰당 30달러입니다.

실전 팁: 7월 19일 이전이 두 모델을 동시 비교하는 비용 대비 최적 구간입니다. 동일한 코딩 태스크를 양쪽에 던져 결과물 품질과 속도를 직접 측정해 두십시오. 프로모션이 끝난 뒤 팀 도구를 결정하면 근거 있는 선택이 됩니다.

## 2. GitHub Actions + uvx: 캐시를 재활용하는 CI/CD 설정법

uvx를 GitHub Actions에서 반복 실행할 때 매번 PyPI에서 패키지를 내려받는 문제가 있습니다. 환경 변수 한 줄로 해결할 수 있습니다. [출처](https://simonwillison.net/2026/Jul/14/uvx-github-actions-cache)

워크플로 YAML에 아래 두 가지를 추가하십시오.

```yaml
env:
  UV_EXCLUDE_NEWER: "2026-07-12"
```

그리고 캐시 키에 해당 날짜 값을 포함시킵니다.

```yaml
key: uvx-${{ env.UV_EXCLUDE_NEWER }}-${{ runner.os }}
```

이 방식은 도구 해석을 특정 날짜에 고정해 동일 날짜 기준으로 캐시를 재사용합니다. 업그레이드가 필요할 때는 날짜 값만 바꾸면 캐시가 자동 무효화됩니다.

실전 팁: linter·formatter처럼 버전 변경이 잦지 않은 도구일수록 효과가 큽니다. CI/CD 실행 시간을 줄이면 강의 실습 환경에서도 대기 시간이 체감될 정도로 단축됩니다.

## 3. Claude Code 브라우저 인터페이스·Cursor 범용 에이전트: 지금 바로 쓸 수 있는 기능

**Claude Code 브라우저 인터페이스**가 공개됐습니다. 로컬 환경 설치 없이 웹에서 코드를 실행하고 테스트할 수 있습니다. [출처](https://tldr.tech/ai/2026-07-13)

강의 시연이나 팀원과의 예제 공유에 즉시 활용할 수 있습니다. 수강생 환경 세팅 문제를 우회하는 용도로도 유용합니다.

**Cursor**는 범용 에이전트(General Agent) 기능을 추가했습니다. 복잡한 다단계 태스크를 에이전트에 맡기면 코드베이스 전체를 분석·수정·실행하는 계획을 자동으로 수립합니다.

**Claude Fable 확장**은 타임트래블 디버깅(Time-travel Debugging)을 제공합니다. 실행 흐름을 역방향으로 추적해 버그 발생 시점의 상태를 그대로 확인할 수 있습니다. 전통적인 전진 방식 디버깅보다 원인 파악 속도가 빠릅니다.

실전 팁: 브라우저 인터페이스와 타임트래블 디버깅은 강의 라이브 코딩 장면에서 특히 강점이 드러납니다. 환경 오류 없이 버그를 재현하고 역추적하는 과정을 그대로 보여줄 수 있습니다.

## 4. Codex 700만 사용자 vs Claude Code: 수치 이면의 구도

**OpenAI Codex**의 사용자 수가 6개월 만에 10배 이상 늘어 약 700만 명에 달한다는 보고가 나왔습니다. 하루 사이 100만 명이 추가됐다는 주장도 함께 제기됐습니다. [출처](https://www.latent.space/p/ainews-codex-usage-up-10x-in-6-months)

다만 이 수치의 집계 방식과 근거는 공개되지 않았습니다. Claude Code는 같은 기간 정기적인 사용량 데이터를 발표하지 않았습니다.

과장 vs 실속 구도를 정리하면 이렇습니다. 사용자 수 급증 자체는 Codex의 접근성 확대를 반영하는 신호입니다. 그러나 실무 활용 가치를 판단하려면 TerminalBench 같은 벤치마크 성능, API 단가, 워크플로 연동성을 함께 봐야 합니다.

오늘 도구 흐름의 핵심은 단순합니다. ChatGPT와 Claude가 한도를 풀고 기능을 추가하는 지금이 비용 부담 없이 두 도구를 나란히 검증할 수 있는 구간입니다. 수치 경쟁보다 자신의 실무 태스크에서 직접 측정한 결과가 더 신뢰할 수 있는 판단 기준입니다.
