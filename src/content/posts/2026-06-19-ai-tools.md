## 1. Datasette Apps 0.1a2: 샌드박스 앱 호스팅 + CSP 권한 제어

**Datasette Apps** 플러그인이 2026년 6월 공개되었습니다. [출처](https://simonwillison.net/2026/Jun/18/datasette-apps)

핵심 구조는 `iframe sandbox="allow-scripts allow-forms"` 안에 HTML·CSS·JavaScript 앱을 올리는 방식입니다.

앱 코드는 클라이언트 사이드에서 Datasette의 SQL 쿼리를 직접 호출합니다. 쿠키·`localStorage` 접근은 차단되고, 외부 HTTP 요청도 주입된 콘텐츠 보안 정책(Content Security Policy, CSP) 헤더로 막힙니다.

**쓰는 법은 다음과 같습니다.**

읽기 전용 대시보드라면 HTML 파일 하나로 충분합니다. 쓰기가 필요하면 플러그인 설정에서 특정 쿼리에 write 권한을 부여하면 됩니다.

0.1a2에서 추가된 `apps-set-csp` 권한은 비관리자도 허용된 CSP 오리진(origin)을 설정할 수 있게 합니다. `allowed_csp_origins` 허용 목록을 플러그인 설정에 정의하면, 신뢰된 사용자에게만 범위를 열어줄 수 있습니다. [출처](https://simonwillison.net/2026/Jun/15/datasette-apps)

**실전 팁:** 타임라인·탐색형 대시보드·간단한 관리 화면을 만들 때 Flask나 FastAPI 없이 Datasette 하나로 끝낼 수 있습니다. 멀티테넌트 환경에서는 `allowed_csp_origins`로 팀별 오리진을 격리하는 방식이 권장됩니다.

강의 시연에서는 "백엔드 없는 데이터 앱" 패턴으로 소개하면 Datasette의 진입 장벽이 낮아지는 효과가 있습니다.

## 2. ChatGPT Enterprise: 부서별 예산 한도·실시간 사용량 분석

OpenAI가 **ChatGPT Enterprise** 관리자 콘솔에 지출 통제 기능을 추가했습니다. [출처](https://openai.com/index/chatgpt-enterprise-spend-controls)

관리자는 부서·팀 단위로 예산 한도를 설정하고, 사용량이 임계치에 근접하면 알림을 받을 수 있습니다.

대시보드에서는 팀·사용자·기능별로 토큰 소비 내역을 실시간으로 확인할 수 있습니다.

**실전 팁:** 전사 접근을 차단하지 않고 팀별 한도를 분리하는 방식으로 거버넌스를 구성할 수 있습니다. AI 도입 초기에 예산 초과를 우려하는 조직이라면, 이 기능으로 사용량 근거를 경영진에게 보여주는 자료로도 활용 가능합니다.

## 3. 구글 OKF: 에이전트 간 지식 공유 표준

구글이 **오픈 지식 포맷(Open Knowledge Format, OKF)** v0.1을 공개했습니다. [출처](https://news.hada.io/topic?id=30622)

여러 AI 에이전트가 서로 다른 생산자가 만든 위키를 별도 변환 없이 소비할 수 있도록 설계된 벤더 중립 규격입니다.

형식은 단순합니다. YAML 프런트매터(frontmatter)와 마크다운을 조합한 구조로, 기존 LLM 위키 패턴을 공식화한 것입니다.

**쓰는 법:** 팀 내부 문서를 OKF 형식으로 작성해 두면, Claude·Gemini·사내 에이전트 등 여러 시스템에서 재포맷 없이 바로 읽을 수 있습니다.

**실전 팁:** 멀티 에이전트 시스템을 설계 중이라면 지식 베이스를 OKF로 통일해 두는 것이 벤더 교체 비용을 줄이는 구조적 선택입니다.

## 4. AI 토큰 비용 절감 전략: 캐시·라우팅·프롬프트 최적화

토큰 비용 증가에 대응하는 실무 전략 세 가지가 정리되었습니다. [출처](https://www.itworld.co.kr/article/4187067/)

첫째, **모델 라우팅**입니다. 복잡도가 낮은 작업은 Gemini Flash처럼 저비용 모델로 먼저 보내고, 추론이 필요한 작업만 고사양 모델로 넘깁니다.

둘째, **메모리 레이어 캐시**입니다. DevRev 사례처럼 지식 그래프를 CPU 캐시로 유지하면, GPU 토큰 소비를 줄이고 응답 속도를 확보할 수 있습니다.

셋째, **프롬프트 엔지니어링**입니다. 쿼리를 구체화하면 불필요한 컨텍스트를 줄여 토큰 효율을 높입니다. 인프라 변경 없이 당장 적용 가능한 유일한 수단입니다.

**실전 팁:** 세 전략을 동시에 도입하기 어렵다면 프롬프트 최적화부터 시작하는 것이 가장 빠른 비용 절감 경로입니다.

## 5. 오늘의 흐름 정리: 구조 비용과 표준화

오늘 신호들은 두 축으로 수렴합니다.

하나는 **비용 구조화**입니다. 토큰 비용이 오를수록 라우팅·캐시·권한 분리처럼 아키텍처 레벨의 설계가 경쟁력이 됩니다.

다른 하나는 **표준화**입니다. OKF처럼 에이전트 간 지식 형식을 통일하려는 시도가 본격화되고 있습니다.

Datasette Apps는 이 두 축의 교차점에 있습니다. 별도 서비스 없이 샌드박스로 앱을 격리하는 패턴은 비용과 보안을 동시에 잡는 구조적 선택입니다. Noam Shazeer의 OpenAI 합류는 모델 아키텍처 경쟁이 수면 위로 올라온 신호로 읽힙니다. [출처](https://news.hada.io/topic?id=30626)
