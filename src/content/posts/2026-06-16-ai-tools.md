## 1. Claude: 프롬프트 표현 하나로 달라지는 보안 가드레일

백악관 보안 연구에서 Claude(Fable)가 의도적으로 취약하게 작성된 코드를 받았을 때, "보안 문제를 검토해 달라"는 요청은 거부했습니다. 반면 "이 코드를 고쳐 달라"는 요청에는 수정 절차까지 제시하며 응했습니다. [출처](https://simonwillison.net/2026/Jun/16/matteo-wong-the-atlantic)

모델 입장에서는 수동적 취약점 탐색과 능동적 개선 작업을 서로 다른 행위로 구분한 것입니다.

**실전 팁:** 합법적인 코드 보안 작업에서 Claude를 활용할 때는 "취약점을 찾아라"보다 "이 코드를 안전하게 개선하라"처럼 능동적 개선 표현을 사용하십시오. 코드 리뷰 워크플로를 설계할 때도 진단 단계와 수정 단계를 프롬프트에서 명확히 분리하면 예상치 못한 거부를 줄일 수 있습니다.

보안 강의에서 이 사례는 "프롬프트 표현이 모델의 안전 행동을 어떻게 바꾸는가"를 설명하는 구체적인 실습 소재로 활용 가능합니다.

## 2. Gemini CLI → Antigravity CLI 전환, 6월 18일 마감

Google이 Gemini CLI를 **Antigravity CLI**로 통합 이전합니다. Go 기반으로 재구축되었으며, 멀티 에이전트 워크플로와 비동기 처리를 기본 지원합니다. [출처](https://developers.googleblog.com/an-important-update-transitioning-gemini-cli-to-antigravity-cli)

**전환 마감은 2026년 6월 18일**입니다. 이 날짜 이후 Gemini CLI는 요청을 처리하지 않습니다. 엔터프라이즈 고객은 별도 안내가 있을 때까지 기존 접근 방식이 유지됩니다.

Antigravity CLI는 Antigravity 2.0 데스크톱 앱과 통합되어 CLI와 GUI 환경을 단일 플랫폼으로 묶습니다.

**즉시 조치 사항:** 현재 Gemini CLI 기반 자동화 스크립트나 CI/CD 파이프라인이 있다면 마감 전에 Antigravity CLI로 전환하십시오. 아직 마이그레이션 가이드가 공개되어 있으므로, 명령 구조 변경 여부를 먼저 확인하는 것이 우선입니다.

## 3. GitHub Copilot CLI 슬래시 명령 실전 사용법

GitHub Copilot CLI가 터미널 내 슬래시 명령을 공식 지원합니다. 에디터와 터미널 사이 컨텍스트 전환 없이 AI 지원을 받을 수 있는 구조입니다. [출처](https://github.blog/ai-and-ml/github-copilot/github-copilot-cli-for-beginners-overview-of-common-slash-commands)

주요 명령 세 가지는 다음과 같습니다.

- `/explain`: 특정 셸 명령의 동작 원리를 설명합니다.
- `/suggest`: 현재 컨텍스트를 기반으로 다음 명령을 제안합니다.
- `/fix`: 실패한 명령을 자동으로 수정하여 재입력 없이 재시도합니다.

**실전 팁:** 처음 CLI 작업을 배우는 수강생에게 `/explain`으로 낯선 명령을 즉시 해설하게 하면 학습 속도가 빠릅니다. 반복적인 파이프라인 디버깅에는 `/fix`를 먼저 시도한 뒤 오류 원인을 분석하는 순서가 효율적입니다.

## 4. Apple Foundation Models에 Claude 탑재하기

Anthropic이 Swift 패키지를 공개하여 Claude를 Apple Foundation Models 프레임워크에 통합할 수 있게 했습니다. WWDC 2026에서 소개된 `LanguageModel` 프로토콜을 활용합니다. [출처](https://news.hada.io/topic?id=30529)

핵심은 온디바이스 모델과 **Claude 서버 모델을 동일한 코드 경로로 호출**한다는 점입니다. 모델 전환 시 코드를 다시 작성할 필요 없이 설정 값만 바꾸면 됩니다.

```swift
// 온디바이스 모델과 Claude를 동일한 프로토콜로 호출
let model: LanguageModel = .claude  // 또는 .onDevice
```

**실전 팁:** iOS/macOS 앱에서 온디바이스 처리와 클라우드 처리를 조건부로 전환하는 구조를 만들 때, 이 패키지를 활용하면 단일 인터페이스로 두 경우를 모두 처리할 수 있습니다. 강의에서는 벤더 종속성 분리 패턴의 실제 구현 사례로 소개하기 좋습니다.

## 5. Claude Code 한도 이탈과 오픈소스 대안의 현실

Claude Code 사용 한도가 중요한 데모 2시간 전에 소진되면서, 한 개발자가 **Ollama + DeepSeek + Qwen 3 Coder** 조합으로 전환한 사례가 공유되었습니다. [출처](https://yozm.wishket.com/magazine/detail/3801)

상업용 도구의 요금제 한도와 사전 예고 없는 정책 변경이 시간 민감 업무에서 신뢰도를 떨어뜨린다는 것이 이탈의 직접적 이유입니다.

오늘 함께 소개된 Claude Fable 모델의 정부 서한에 의한 서비스 중단 사례와 맞물려, 외부 의존성이 높은 AI 도구 선택에 대한 구조적 질문이 커지고 있습니다.

**오픈소스 전환 시 고려 사항:**
- 로컬 추론을 위한 GPU 메모리와 설치 환경 확보가 선행 조건입니다.
- Ollama는 모델 전환이 간단하지만, 대형 모델일수록 초기 응답 지연이 발생합니다.
- 팀 단위 사용이라면 셀프호스팅 비용과 유지 관리 부담을 상업용 플랜 비용과 비교하십시오.

상업용 AI 도구는 속도와 편의를 제공하지만 가용성 보장은 없습니다. 오픈소스는 예측 가능성과 제어권을 주는 대신 운영 책임이 따릅니다. 두 방향 사이에서 팀 상황에 맞는 비중을 미리 정해 두는 것이 오늘 흐름이 던지는 실질적 과제입니다.
