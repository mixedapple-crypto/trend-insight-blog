## 1. Claude Sonnet 5: 신규 토크나이저와 실제 비용 계산

**Claude Sonnet 5**가 2026년 6월 30일 공개됐습니다.

공식 가격은 입력 $3·출력 $15(백만 토큰 기준)로, 이전 Sonnet 4.6 대비 인하처럼 보입니다. 그런데 신규 토크나이저가 영어 텍스트를 약 1.42배 더 많은 토큰으로 분할합니다.

실질 비용은 기존 대비 거의 동일하거나 오히려 높을 수 있습니다.
언어별 팽창률은 영어 1.42배, Python 코드 1.28배, 간체 중국어 1.01배입니다.

**실전 팁:** Anthropic이 제공하는 Claude Token Counter 도구로 현재 프롬프트를 직접 측정하십시오. 마이그레이션 전 반드시 비교해야 합니다.

API 변경 사항도 체크해야 합니다. `temperature`, `top_p`, `top_k` 샘플링 파라미터가 더 이상 지원되지 않습니다. 어댑티브 씽킹(Adaptive Thinking)은 기본 활성화 상태입니다.

에이전트 실행 능력은 기존 Opus급 수준에 가깝고, 에포트(effort) 컨트롤로 태스크별 비용·성능 비율을 조정할 수 있습니다. 소개 할인($2/$10)은 8월 31일까지 적용되므로, 이 기간을 벤치마킹 창으로 활용하십시오. [출처](https://simonwillison.net/2026/Jun/30/claude-sonnet-5)

## 2. Microsoft Memora: 에이전트 메모리 토큰 98% 절감

Microsoft Research가 에이전트(Agent) 메모리 시스템 **Memora**를 공개했습니다.

기존 벡터 스토어 방식은 유사도 기반 검색에 강하지만, "무엇이 바뀌었는가"와 "지금 유효한 정보인가"를 구분하지 못합니다. Memora는 저장과 검색을 분리하는 구조로 이 문제를 해결합니다.

구조는 세 레이어로 구성됩니다.

- 추상(Abstraction): 6~8단어 요약 키
- 메모리 값(Memory Value): 전체 맥락 정보
- 큐 앵커(Cue Anchor): 의미 태그 기반 검색 트리거

동일 추상 키 아래 업데이트를 병합해 단편화를 방지하고, 폴리시 기반 리트리버(Policy-guided Retriever)가 쿼리를 반복 정제합니다. 그 결과 컨텍스트 토큰 소비를 최대 98% 줄이면서 정확도를 유지합니다.

주간 단위 이상 대화를 처리하는 장기 에이전트나, 기업 환경의 멀티 에이전트 파이프라인에 바로 적용할 수 있는 설계입니다. [출처](https://www.itworld.co.kr/article/4191363/%ed%86%a0%ed%81%b0-%ec%86%8c%eb%b9%84-%ec%b5%9c%eb%8c%80-98-%ec%a0%88%ea%b0%90-%eb%a7%88%ec%9d%b4%ed%81%ac%eb%a1%9c%ec%86%8c%ed%94%84%ed%8a%b8-ai-%ec%97%90%ec%9d%b4%ec%a0%84%ed%8a%b8-%eb%a9%94.html)

## 3. DESIGN.md: AI 코드 생성에 디자인 시스템 고정하기

Atlassian이 **DESIGN.md** 형식을 공개했습니다.

디자인 토큰, 컴포넌트 규칙, 브랜드 가이드라인을 하나의 마크다운 파일로 묶어 AI 프롬프트에 포함하는 방식입니다. Claude, ChatGPT 등에 UI 생성을 요청할 때 매번 브랜드 맥락을 재설명하는 비효율을 없앱니다.

**사용법:** 프로젝트 루트에 `DESIGN.md`를 두고, UI 관련 프롬프트 첫 줄에 파일 내용을 붙여 넣거나 시스템 프롬프트로 고정합니다. 이후 AI는 커스텀 디자인 시스템 규칙을 전제로 코드를 생성합니다.

실전 효과는 반복 수정 횟수 감소입니다. "브랜드 컬러 적용해 줘"처럼 모호한 지시 대신, 문서화된 토큰 값이 출력을 고정합니다. 강의 환경에서는 DESIGN.md를 실습 재료로 제공해 AI 코드 품질 편차를 줄이는 용도로도 쓸 수 있습니다. [출처](https://news.hada.io/topic?id=30990)

## 4. Claude Code 스테가노그래피: 프록시 우회 시 주의

**Claude Code 2.1.196** 바이너리 분석에서 주목할 만한 동작이 발견됐습니다.

환경 변수 `ANTHROPIC_BASE_URL`이 공식 `api.anthropic.com`이 아닌 엔드포인트로 설정되면, 시스템 프롬프트 내 현재 날짜 문장을 분류 신호로 교체하는 함수가 존재합니다. 이 함수는 호스트명과 타임존 데이터를 함께 확인합니다.

즉, 서드파티 프록시나 커스텀 라우팅을 통해 Claude Code를 사용할 경우 Anthropic 측에서 비공식 경로 사용을 탐지할 수 있습니다.

**실전 적용:** 사내 프록시, 비용 모니터링 레이어, 또는 테스트용 인터셉터를 운용 중이라면 이 동작을 인지하십시오. 프라이버시나 계약 조건 측면에서 추가 검토가 필요합니다. [출처](https://news.hada.io/topic?id=30980)

## 5. 로컬 AI 부상 vs. 클라우드 의존 구조 재점검

오늘 신호를 묶으면 하나의 대비 구조가 보입니다.

클라우드 모델은 Sonnet 5처럼 성능을 높이면서 가격 체계를 복잡하게 만드는 방향으로 움직입니다. 반면 로컬 AI는 모델 압축, 양자화(Quantization), 추론 최적화 덕분에 성능 격차를 빠르게 좁히고 있습니다.

레이턴시, 데이터 레지던시(Data Residency), 운영 비용이 중요한 환경에서는 로컬 모델이 실질적인 대안이 되고 있습니다. Claude Code의 스테가노그래피 논란도 같은 맥락입니다. API 의존 구조의 투명성 문제를 드러내는 사례입니다.

Sonnet 5 토크나이저 함정, Memora의 메모리 분리 설계, DESIGN.md의 컨텍스트 고정 전략은 모두 같은 방향을 가리킵니다. **토큰과 컨텍스트를 얼마나 효율적으로 관리하는가**가 실무 AI 비용의 핵심 변수로 자리 잡고 있습니다. [출처](https://www.latent.space/p/ahmad-osman-local-ai)
