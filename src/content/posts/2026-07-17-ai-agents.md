오늘 AI 업계에서는 **Mira Murati**의 Thinking Machines Lab과 중국의 Moonshot AI가 대형 오픈웨이트 모델을 잇따라 공개하며 가장 굵은 신호를 남겼습니다. Thinking Machines Lab은 975B 파라미터 규모의 멀티모달 모델 Inkling을, Moonshot AI는 2.8조 파라미터의 Kimi K3를 각각 발표했습니다. 그 사이 NVIDIA의 임베딩 모델이 검색 벤치마크 1위를 차지했고, OpenAI의 코드 실행 도구에서 파일 삭제 버그가 보고되는 등 부수 신호도 뚜렷했습니다. 한쪽에서는 오픈소스 생태계가 넓어지고, 다른 한쪽에서는 비감독 코드 실행의 안전 공백이 드러나며 빛과 그림자가 동시에 나타났습니다.

## 1. Thinking Machines Lab의 첫 모델 공개: Inkling, Apache 2.0, 975B

Thinking Machines Lab이 **Inkling**을 공개했습니다. [출처](https://simonwillison.net/2026/Jul/16/inkling)

총 파라미터 975B, 활성 파라미터 41B 규모의 멀티모달(Multimodal) 모델입니다. 텍스트·이미지·오디오·영상을 포함한 45조 토큰으로 사전 학습되었습니다.

라이선스는 Apache 2.0으로, 상업적 활용과 파인튜닝(fine-tuning)이 자유롭습니다. 소형 변형 모델인 Inkling-Small(276B 총 파라미터, 12B 활성)도 테스트 중이며, 가중치는 추후 공개될 예정입니다.

연구소는 이 모델을 최전선 모델이 아닌 파인튜닝용 기반 모델로 규정합니다. 자체 학습 플랫폼 Tinker를 통한 커스터마이징을 주요 활용 경로로 제시했습니다.

NVIDIA Nemotron, 구글 Gemma 4에 이어 미국산 오픈웨이트 생태계에 새 경쟁자가 추가된 셈입니다. Mira Murati가 OpenAI를 떠난 뒤 세운 연구소의 첫 공개 모델이라는 점에서 시장의 주목을 받고 있습니다.

---

## 2. Moonshot AI의 Kimi K3: 2.8조 파라미터, 고가 정책

Moonshot AI가 **Kimi K3**를 발표했습니다. [출처](https://simonwillison.net/2026/Jul/16/kimi-k3)

총 파라미터 2.8조 규모로, 자체 벤치마크에서 Claude Opus 4.8 max와 GPT-5.5 high를 앞섰습니다. Artificial Analysis의 비공개 평가에서는 1547 엘로(Elo)를 기록했습니다.

API 가격은 입력 토큰 100만 개당 **3달러**, 출력 100만 개당 15달러입니다. 중국 AI 연구소 모델 중 가장 높은 가격이며, Anthropic Claude Sonnet과 비슷한 수준입니다.

오픈웨이트 공개는 2026년 7월 27일로 예고되어 있습니다. 단, 자체 보고 벤치마크에서 Claude Fable 5와 GPT-5.6 Sol에는 미치지 못한다고 명시했습니다.

중국 AI 연구소가 프리미엄 가격 정책을 채택한 것은 주목할 만한 변화입니다. 저비용 경쟁이 아닌 성능 포지셔닝으로 전략 축을 옮기는 흐름이 읽힙니다.

---

## 3. NVIDIA Nemotron 3 Embed: RTEB 1위, 에이전트 검색

**NVIDIA**의 Nemotron 3 Embed 모델이 검색 텍스트 임베딩 벤치마크(RTEB, Retrieval Text Embedding Benchmark) 종합 1위에 올랐습니다. [출처](https://huggingface.co/blog/nvidia/nemotron-3-embed-wins-rteb)

모델은 현재 Hugging Face를 통해 공개되어 있습니다. NVIDIA는 이 결과가 시맨틱(Semantic) 검색에 의존하는 에이전트(Agent) 시스템에 직접적인 이점을 가져다준다고 밝혔습니다.

에이전트 워크플로에서 검색 품질은 최종 응답 정확도에 직결됩니다. 임베딩 모델의 성능 경쟁이 에이전트 인프라 전반의 품질을 끌어올리는 구조로 자리 잡고 있습니다.

---

## 4. GPT-5.6 파일 삭제 버그: 비감독 코드 실행 안전 공백

Anthropic 연구원 Thibault Sottiaux가 **GPT-5.6**의 파일 삭제 버그를 보고했습니다. [출처](https://simonwillison.net/2026/Jul/16/bad-codex-bug)

샌드박스 없이 전체 접근 모드가 활성화된 환경에서 발생합니다. 모델이 임시 디렉터리를 설정하려다 `$HOME` 환경 변수 자체를 삭제하는 오류입니다.

자동 검토 없이 비감독 코드 실행을 허용하면 데이터 손실로 이어질 수 있습니다. LLM 기반 코드 실행 환경에서 격리(샌드박싱) 없는 운용이 얼마나 위험한지를 구체적 사례로 확인시켜 주었습니다.

---

## 5. 에이전트 확장 신호들: 지연 최적화·Cars24·DoorDash CLI

에이전트 적용 범위가 연구와 산업 양쪽에서 넓어지는 신호가 이날 여러 건 포착되었습니다.

arXiv에 발표된 LAMaS 프레임워크는 멀티 에이전트 시스템의 추론 지연을 50% 이상 줄이면서 정확도를 유지하는 방법을 제시했습니다. [출처](https://arxiv.org/abs/2607.13359) 실행 경로 전체가 아닌 임계 경로(critical path)만을 최적화 대상으로 삼은 점이 핵심입니다.

중고차 플랫폼 **Cars24**는 OpenAI 기반 음성·채팅 에이전트로 월 100만 분 이상의 대화를 처리하고, 이탈 리드(lead)의 12%를 회수했다고 밝혔습니다. [출처](https://openai.com/index/cars24) 대화형 에이전트가 영업 퍼널을 직접 개선한 사례입니다.

**DoorDash**는 개발자와 AI 에이전트가 터미널에서 주문을 완료할 수 있는 CLI 도구 `dd-cli`의 한정 베타를 시작했습니다. [출처](https://techcrunch.com/2026/07/16/yes-you-can-now-order-doordash-from-the-command-line) 플랫폼이 사람이 아닌 에이전트를 1차 인터페이스 대상으로 설계하기 시작했다는 점에서, 소비자 서비스가 에이전트 소비 구조로 전환되는 흐름을 보여줍니다.

이날 하루는 대형 모델 공개가 오픈 생태계를 넓힌 반면, GPT-5.6 버그가 비감독 실행의 위험을 재확인하는 구도를 만들었습니다. 인프라·산업 현장 전반에서 에이전트 채택이 가속되는 속도만큼, 그 실행 환경의 안전 설계가 뒤따라야 한다는 과제가 선명해진 날이었습니다.
