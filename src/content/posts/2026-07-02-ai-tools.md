## 1. @ttsc/graph MCP: Claude Code 타입스크립트 분석 토큰 10배 절감

**@ttsc/graph**는 타입스크립트(TypeScript) 컴파일러의 사전 파싱 코드 그래프를 MCP(Model Context Protocol)로 노출하는 오픈소스 도구입니다.

기존 Claude Code는 코드베이스를 분석할 때 파일을 반복 열고 임포트를 수동으로 추적합니다. 이 과정에서 grep 루프가 반복되며 토큰을 대량 소모합니다.

@ttsc/graph를 적용하면 코드베이스를 한 번만 통과시켜 호출 의존성·함수 호출·임포트 관계를 그래프로 구축합니다. 이후 Claude Code에 "X는 어떻게 동작하는가?"라고 질문하면 그래프를 직접 참조해 파일 재독 없이 답합니다.

실전 팁은 두 가지입니다.

첫째, 대규모 모노리포(monorepo)나 처음 접하는 코드베이스에서 효과가 가장 큽니다. 파일 수가 많을수록 절감 폭이 커집니다.

둘째, 구조 파악 질문("이 함수를 호출하는 곳은?")에 특화되어 있습니다. 전체 파일 내용이 필요한 작업과 역할을 나눠 쓰는 것이 적합합니다.

강의 시연에서는 "같은 질문을 그래프 없이/있이 각각 던져 토큰 카운트를 비교"하는 방식으로 효과를 직접 보여줄 수 있습니다. [출처](https://news.hada.io/topic?id=31006)

## 2. CTX v0.3.40: Claude Code 세션 간 영구 메모리

**CTX v0.3.40**은 Claude Code 세션 사이에 대화 이력·결정 사항·파일 참조를 자동으로 이어주는 지속 메모리(persistent memory) 도구입니다.

설치는 `pip install ctx`로 끝납니다. 설치 후 CTX는 `UserPromptSubmit` 훅을 통해 이전 세션의 컨텍스트를 새 세션 시작 시 자동 주입합니다.

이전까지는 세션이 끊길 때마다 "이 파일의 구조는 이렇고, 지난번에 이런 결정을 했다"고 다시 설명해야 했습니다. CTX는 이 반복을 제거합니다.

실전 팁입니다. 장기 프로젝트나 여러 날에 걸친 반복 개발에서 특히 유용합니다. 세션 재시작 시 코드베이스 재설명 비용이 사라지므로, 실질 토큰 절감 효과도 따라옵니다.

현재 월 2,726 다운로드·98명 사용자 규모로, 초기 채택 단계입니다. [출처](https://news.hada.io/topic?id=31000)

## 3. shot-scraper 1.10: 에이전트 작업을 영상으로 자동 기록

**shot-scraper 1.10**은 브라우저 인터랙션을 YAML 스토리보드로 정의하고, 에이전트가 자신의 작업 과정을 자동으로 영상 파일로 기록하게 해주는 커맨드라인 도구입니다.

핵심 명령은 아래와 같습니다.

```
shot-scraper video storyboard.yml --auth credentials.json --mp4
```

`storyboard.yml`에 서버 시작, 뷰포트 크기, 클릭·입력·대기 등 UI 동작 시퀀스를 정의합니다. 도구가 순서대로 실행하며 MP4로 저장합니다.

클립보드 데이터 목킹, 엘리먼트 대기, 실제 사용자 워크플로 시뮬레이션도 지원합니다. 수동 녹화 없이 기능이 엔드투엔드(end-to-end)로 동작함을 증명하는 용도에 적합합니다.

강의나 개발 문서화에서 "에이전트가 스스로 데모 영상을 만든다"는 구조는 반복 시연 비용을 크게 줄여줍니다. [출처](https://simonwillison.net/2026/Jun/30/shot-scraper-video)

## 4. 모델 라우팅(토큰매칭): AI 비용 절반 줄이는 구조 전환

"토큰매킹(tokenmaxxing, 토큰 최대 소모)"에서 **토큰매칭(tokenmatching, 작업별 모델 최적화)**으로의 전환이 비용 전략의 핵심 축으로 부상하고 있습니다.

Coinbase는 단순 쿼리에 저비용 모델을, 코드 리뷰 같은 복잡 작업에 프론티어 모델을 배분해 비용을 절반으로 줄였습니다. 오픈소스 도구 **Claude Code Router**는 프롬프트를 자동 분석해 비용 적합 모델로 라우팅합니다.

실전 팁입니다. 작업을 두 등급으로 나누는 것에서 시작합니다.

- 단순 쿼리·포맷 변환·요약 → 경량 모델
- 코드 리뷰·아키텍처 설계·복잡한 추론 → 프론티어 모델

팀 단위로 도입하면 품질 저하 없이 비용을 30~50% 절감할 수 있다는 사례가 보고되고 있습니다. [출처](https://www.itworld.co.kr/article/4191945/ai-%eb%b9%84%ec%9a%a9-%ec%a0%88%eb%b0%98%ec%9c%bc%eb%a1%9c-%ec%a4%84%ec%9d%b4%eb%8a%94-%eb%aa%a8%eb%8d%b8-%eb%9d%bc%ec%9a%b0%ed%8c%85-%ec%9d%b4%ec%a0%9c%eb%8a%94-%ed%86%a0%ed%81%b0%eb%a7%a4%ec%b9%ad.html)

## 5. Cursor iOS 앱: 프라이버시 설정 비가역 변경 경보

**Cursor** iOS 앱을 설치하고 로그인하면, Cloud Agents 설정 과정에서 계정의 프라이버시 모드가 자동으로 다운그레이드됩니다.

구체적으로, 코드 저장을 막는 "Privacy Mode (Legacy)"에서 백그라운드 에이전트용 저장을 허용하는 "Privacy Mode"로 전환됩니다. 이 변경은 앱 내에서 되돌릴 수 없고, 지원팀에 직접 연락해야만 복구를 시도할 수 있습니다.

문제는 이 과정에서 결과를 명확히 설명하는 안내가 없다는 점입니다. 선택 취소 옵션도 제공되지 않습니다.

대응 방법은 명확합니다. iOS 앱 설치 전에 데스크톱 계정의 현재 프라이버시 설정을 반드시 확인합니다. 코드 비저장 정책이 필요한 환경이라면 iOS 앱 설치 자체를 보류합니다.

오늘 신호 전체를 한 줄로 묶으면, 비용·컨텍스트·기록·보안 네 축에서 실무 마찰을 줄이는 도구들이 동시에 움직였습니다. 새 기능 채택과 설정 검증을 함께 챙겨야 하는 날입니다. [출처](https://news.ycombinator.com/item?id=48737226)
