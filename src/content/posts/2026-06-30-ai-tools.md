## 1. 클로드 코드 창시자의 5가지 실무자 원형

클로드 코드 창시자 보리스 체르니(Boris Cherni)가 AI 시대 실무자를 다섯 원형으로 제시했습니다. [출처](https://news.hada.io/topic?id=30965)

**프로토타이퍼(Prototyper)**, 빌더(Builder), 스위퍼(Sweeper), 그로어(Grower), 그리고 다섯 번째 원형입니다.

각 원형이 AI 도구를 쓰는 방식은 구체적으로 다릅니다.

- **프로토타이퍼**: 아이디어 검증에 클로드의 빠른 반복 생성을 활용합니다.
- 빌더: 아키텍처 설계에 집중하고, 보일러플레이트(boilerplate) 코드는 AI에 맡깁니다.
- 스위퍼: 기술 부채 정리를 배치(batch) 자동화로 처리합니다.
- 그로어: 시장 확장 단계에서 AI로 운영 효율을 높입니다.

강의 설계에도 적용할 수 있습니다. 수강생이 어느 원형인지 먼저 파악하면, AI 도구 도입 순서와 깊이를 다르게 구성할 수 있습니다.

## 2. Ornith-1.0: 로컬에서 돌리는 에이전트 코딩 모델

DeepReinforce가 **Ornith-1.0**을 MIT 라이선스로 공개했습니다. 9B~397B 파라미터 규모이며, Gemma 4와 Qwen 3.5 기반입니다. [출처](https://simonwillison.net/2026/Jun/29/ornith)

Simon Willison이 35B 변형을 **LM Studio**로 테스트했습니다. Datasette 저장소에서 특정 코드 섹션을 찾고, 멀티스텝 도구 호출을 연속 실행하는 데 성공했습니다. 속도는 초당 103 토큰입니다.

로컬 실행 방법은 다음과 같습니다.

1. LM Studio에서 `ornith-1.0-35b-Q4_K_M.gguf` 다운로드(20GB)
2. 도구 호출(tool call) 기능을 활성화하고 에이전트 워크플로 구성
3. 클라우드 없이 반복적 코드 탐색·수정 루프 실행

두 베이스 모델 모두 Apache 2.0 라이선스여서 상업용 프로덕션에 제약이 없습니다. 오프라인 환경이나 코드 보안이 중요한 프로젝트에 바로 적용할 수 있습니다.

## 3. HTML 표 추출 도구: 붙여넣기 한 번으로 다섯 포맷 변환

Simon Willison이 공개한 HTML 표 추출 도구는 웹 페이지의 리치 텍스트(rich text)를 붙여넣으면 모든 표를 자동 감지합니다. [출처](https://simonwillison.net/2026/Jun/29/html-table-extractor)

출력 포맷은 HTML, 마크다운(Markdown), **CSV**, TSV, JSON 다섯 가지입니다.

위키피디아(Wikipedia) 검색 기능도 내장되어 있습니다. CORS API를 통해 페이지를 직접 불러오므로, 브라우저에서 복사 없이 표를 추출할 수 있습니다.

데이터 분석 강의에서 "웹 데이터 → 구조화 데이터" 흐름을 시연할 때 유용합니다. 별도 스크립트 없이 단계를 한 번에 보여줄 수 있습니다.

## 4. 에러를 클로드 코드에 그대로 붙여넣으면 안 되는 이유

스택 트레이스(stack trace) 전체를 복사해 클로드 코드에 넣으면, 모델이 노이즈에 묻혀 핵심 문제를 놓칩니다. [출처](https://home.robusta.dev/blog/you-really-shouldnt-copy-paste-errors-into-claude-code)

대신 세 섹션으로 나눠 입력하는 방식이 더 정확한 디버깅 결과를 냅니다.

1. **무슨 일이 일어났는가**: 상황 설명 2~3문장
2. 관련 코드 스니펫(snippet): 문제가 발생한 부분만
3. 에러 메시지: 핵심 한 줄만

구조가 명확하면 클로드가 원인 분리를 먼저 시도합니다. 긴 로그를 통째로 넣을 때보다 수정 제안의 정확도가 높아집니다.

강의에서 "프롬프트 구조화"를 가르칠 때 디버깅 세션을 예시로 쓰기 좋습니다.

## 5. 포드 vs 앤트로픽: 실패와 탈취가 드러낸 구조적 위험

AI 도입의 대비가 뚜렷한 두 사례가 같은 날 나왔습니다.

**포드**는 AI 품질 검사 시스템이 결함을 충분히 잡지 못하자 베테랑 엔지니어 350명을 재고용했습니다. [출처](https://www.itworld.co.kr/article/4190818/%ed%8f%ac%eb%93%9c-ai-%ed%92%88%ec%a7%88-%ea%b2%80%ec%82%ac-%ec%8b%a4%ed%8c%a8-%ec%9d%b8%ec%a0%95%ed%95%98%ea%b3%a0-%eb%b2%a0%ed%85%8c%eb%9e%91-%ec%97%94%ec%a7%80%eb%8b%88%ec%96%b4-350%eb%aa%85.html)

핵심 원인은 도메인 지식 단절입니다. 2020년 이후 5,300명을 감원하면서 AI 학습 데이터로 포착했어야 할 숙련 지식이 함께 사라졌습니다. 포드 부사장은 "설계 요건을 AI에 입력하면 고품질 제품이 나온다고 잘못 믿었다"고 인정했습니다.

결과는 역설적입니다. 사람과 AI를 병행하자 JD Power 2026 초기 품질 조사에서 주류 브랜드 1위(16년 만의 최고)를 기록했습니다.

**앤트로픽** 쪽은 공세적 신호입니다. 알리바바가 2만 5,000개의 가짜 계정으로 클로드에서 2,880만 건의 응답을 추출해 자사 모델을 학습시켰다고 지목했습니다. [출처](https://www.itworld.co.kr/article/4190778/%ec%97%ad%eb%8c%80-%ec%b5%9c%eb%8c%80-%ea%b7%9c%eb%aa%a8-ai-%ed%83%88%ec%b7%a8-%ec%a3%bc%ec%9e%a5%ec%95%a4%ed%8a%b8%eb%a1%9c%ed%94%bd-%ec%95%8c%eb%a6%ac%eb%b0%94%eb%b0%94-%ec%a7%81.html)

이 기법을 증류(distillation)라 합니다. 상위 모델 출력으로 하위 모델을 학습시켜 기능을 복제하는 방식입니다.

실무자 관점에서 점검할 항목은 세 가지입니다.

- API 사용에 인증 계정과 사용량 모니터링 적용
- 계약서에 증류 금지 조항 명시
- 외부 AI 서비스 공급망의 탈취 감지 능력 확인

오늘 두 사례는 같은 구조를 반대 방향에서 보여줍니다. AI를 과신해 사람 지식을 제거하면 빈자리가 생기고, 그 빈자리를 노리는 추출 공격은 이미 대규모로 작동 중입니다.
