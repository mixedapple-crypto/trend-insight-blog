## 1. GLM-5.2: 753B 오픈 웨이트 모델, 프런트엔드 코딩 실전 투입

Z.ai가 6월 16일 **GLM-5.2**를 MIT 라이선스로 공개했습니다. 파라미터(Parameter) 수는 753B이며, 컨텍스트(Context) 창은 100만 토큰으로 GLM-5.1 대비 5배 확장됐습니다.

Artificial Analysis 인텔리전스 인덱스 오픈 모델 부문 1위, Code Arena 웹개발 리더보드 2위(1위는 Claude Fable 5)를 기록했습니다. 이미지 입력 없이 텍스트만으로 이 순위를 낸 점이 구조적으로 눈에 띕니다.

**비용**은 OpenRouter 기준 입력 140만 원/$1.40, 출력 440만 원/$4.40(100만 토큰당)입니다. GPT-5.5나 Claude Opus 4.5~4.8보다 저렴한 구간입니다.

단, 태스크당 출력 토큰이 약 43,000개로 동급 모델보다 높습니다. 장황한 출력을 줄이려면 프롬프트에 응답 형식과 분량 제한을 명시적으로 지정하는 것이 효과적입니다.

실전 투입 방법은 간단합니다. OpenRouter에서 `z-ai/glm-5.2`를 선택하면 기존 OpenAI 호환 API 코드를 그대로 사용할 수 있습니다. 프런트엔드 컴포넌트 생성이나 긴 코드베이스 리뷰처럼 컨텍스트가 많이 필요한 작업에 우선 투입해 보십시오.

강의 시연용으로도 적합합니다. 상용 모델 대비 비용이 낮고, MIT 라이선스라 코드 예제 재배포 제약도 없습니다. [출처](https://simonwillison.net/2026/Jun/17/glm-52)

## 2. Anthropic 서울 사무소 공식 개소와 국내 파트너십

6월 17일 Anthropic이 서울 사무소를 공식 열었습니다. 도쿄·방갈로르에 이은 아시아태평양 세 번째 거점입니다.

한국 시장을 선택한 근거는 수치로 드러납니다. 전 세계 116개국 중 한국의 1인당 **Claude** 사용량이 글로벌 평균의 3.5배입니다.

파트너십은 네이버, 넥슨, 삼성SDS, LG CNS와 체결됐습니다. 엔터프라이즈(Enterprise) 워크플로와 게임 개발 영역에 Claude를 통합하는 방향입니다.

국내 개발자 관점에서는 한국어 지원 품질 개선과 로컬 파트너 API 연동이 가속화될 가능성이 높습니다. Claude를 쓰는 팀이라면 삼성SDS·LG CNS 쪽 엔터프라이즈 계약 경로가 새로 열릴 수 있으니 확인해 두십시오. [출처](https://news.hada.io/topic?id=30600)

## 3. GitHub Copilot 모델 자동 라우팅으로 토큰 절약

GitHub Copilot이 요청 복잡도에 따라 백엔드 모델을 자동 선택하는 라우팅(Routing) 기능을 추가했습니다. 단순 자동완성은 경량 모델로, 복잡한 리팩터링은 대형 모델로 보냅니다.

사용자가 별도 설정할 필요는 없습니다. 요청이 들어오면 Copilot이 내부적으로 판단해 라우팅합니다.

실무 효과는 두 가지입니다. 단순 작업의 지연 시간이 줄고, 월간 토큰 예산이 더 오래 유지됩니다.

팁 하나: 복잡한 설계 질문일수록 컨텍스트를 충분히 붙여야 라우터가 대형 모델로 올바르게 분류합니다. 질문이 짧고 모호하면 경량 모델로 라우팅돼 답변 품질이 낮아질 수 있습니다. [출처](https://github.blog/ai-and-ml/github-copilot/getting-more-from-each-token-how-copilot-improves-context-handling-and-model-routing)

## 4. Datasette 1.0a34: 웹 UI에서 바로 데이터 편집

**Datasette 1.0a34**가 테이블 페이지와 개별 행 페이지에 삽입·수정·삭제 버튼을 추가했습니다. 이전에는 Datasette 에이전트(Agent) 채팅 인터페이스를 통해야만 데이터를 변경할 수 있었습니다.

SQL 없이 웹 UI에서 직접 편집 가능해졌다는 것이 핵심 변화입니다. 비개발자 운영자도 별도 도구 없이 데이터를 관리할 수 있게 됩니다.

강의 환경에서는 수강생에게 SQL 작성 없이 데이터 조작 개념을 먼저 시각적으로 설명하는 데 활용할 수 있습니다. `pip install datasette==1.0a34`로 바로 업그레이드할 수 있습니다. [출처](https://simonwillison.net/2026/Jun/16/datasette)

## 5. "코드는 이제 공짜"—Charity Majors 발언이 남긴 실전 질문

Honeycomb CTO 채리티 메이저스(Charity Majors)의 발언이 6월 17일 Simon Willison 블로그를 통해 퍼졌습니다. 요점은 간단합니다. 2025년 이후 코드 생성 비용이 사실상 0에 수렴했고, 코드 줄 수는 더 이상 아낄 자원이 아닙니다.

대신 희소 자원이 된 것은 테스트, 문서화, 아키텍처(Architecture) 판단력입니다. 잘못 쓴 코드를 유지보수하는 비용이 다시 생성하는 비용보다 비쌉니다.

실전 전환점은 여기 있습니다. AI가 생성한 코드를 배포 전 검증하는 게이트를 얼마나 촘촘하게 설계했는지가 팀 품질을 가릅니다.

이 구조 변화는 오늘 신호들과 맞닿아 있습니다. GLM-5.2처럼 저렴하고 빠른 모델이 늘수록, 코드 생산량이 아니라 검증 설계 역량이 실무자를 구분하는 기준이 됩니다. [출처](https://simonwillison.net/2026/Jun/17/charity-majors)
