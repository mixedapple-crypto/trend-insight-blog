## 1. Kimi K3의 프롬프트 인젝션 방어 패턴

**Kimi K3**는 시스템 프롬프트(System Prompt) 추출 시도나 탈옥(Jailbreak) 요청에 대해 거부 대신 질문으로 전환하는 가드레일(Guardrail)을 탑재했습니다. [출처](https://simonwillison.net/2026/Jul/17/kimi-k3)

"실제로 도움이 필요한 것이 있으신가요?"처럼 재질문 형태로 대화를 유지하면서 모델 내부는 보호합니다.

**실전 팁:** 모델 안전성을 평가할 때, 단순 거부 응답만 체크하는 테스트 케이스는 이 패턴을 통과로 오인할 수 있습니다. 거부 여부가 아니라 응답 구조 자체를 분석하는 테스트 시나리오를 추가해야 합니다.

강의에서 AI 보안을 다룰 때, "거부(Refuse)"와 "우회(Redirect)"가 사용자 경험에서 어떻게 다르게 보이는지 비교 사례로 쓸 수 있습니다.

## 2. LLM 클리셰 하이라이터: AI 투성이 문체 자동 감지

Simon Willison이 **Fable 5**로 제작한 LLM 클리셰(Cliché) 하이라이터가 공개되었습니다. "No fluff, no filler, no jargon" 같은 AI 특유의 반복 표현 열 가지를 자동 감지해 강조 표시합니다. [출처](https://simonwillison.net/2026/Jul/17/llm-cliche-highlighter)

**사용법:**

- 텍스트를 붙여넣으면 클리셰 구문을 시각적으로 하이라이트 처리합니다.
- 편집자·강의 자료 제작자·LLM 출력물 검수 담당자가 수동 검토 없이 패턴을 빠르게 식별할 수 있습니다.

**실전 팁:** LLM 출력물을 그대로 콘텐츠에 쓰는 워크플로라면, 이 도구를 편집 단계 전에 먼저 통과시키는 검수 스텝으로 넣을 수 있습니다.

## 3. Mermaid-ASCII: 브라우저에서 다이어그램을 텍스트로

Go 라이브러리 **mermaid-ascii**가 웹어셈블리(WebAssembly)로 컴파일되어 브라우저에서 직접 실행 가능해졌습니다. [출처](https://simonwillison.net/2026/Jul/16/mermaid-ascii)

Mermaid 다이어그램을 컬러 지원 ASCII 아트로 변환합니다. 서버 처리 없이 클라이언트에서 완결됩니다.

구현 방식은 두 가지입니다. Rust 기반 경량 버전과, 컬러 지원이 포함된 Go 버전(`AlexanderGrooff/mermaid-ascii`) 중 선택합니다.

**활용 시나리오:**

- 터미널 출력에 다이어그램을 포함시켜야 하는 CLI 도구
- Mermaid 렌더링이 불가한 환경의 기술 문서
- 강의 자료에서 텍스트 형태 다이어그램이 필요한 경우

## 4. OpenAI 스코어카드: AI 도입 성과를 4가지로 측정하기

OpenAI CFO Sarah Friar가 AI 도입 ROI(투자 대비 효과) 측정 프레임워크를 발표했습니다. [출처](https://openai.com/index/a-scorecard-for-the-ai-age)

측정 기준 네 가지는 아래와 같습니다.

| 항목 | 측정 내용 |
|---|---|
| 유용한 작업 완료량 | 실제 비즈니스 산출물 |
| 작업당 비용 | 효율성 지표 |
| 신뢰성(Dependability) | 가동률·안정성 |
| 컴퓨트 대비 수익 | 인프라 투자 정당성 |

**실전 팁:** 토큰 수·API 호출 횟수 같은 지표 대신 이 네 항목을 스프린트 리뷰나 강의 평가 기준으로 전환하면, AI 도구 도입 효과를 숫자로 설명하기가 훨씬 쉬워집니다.

## 5. 과대와 실속 사이: 자율 뮤직비디오 실험이 보여주는 것

**Claude Fable 5**와 **GPT-5.6 Sol**이 각각 동일한 곡·가사·예산(25~100달러)을 받아 웹 검색과 ffmpeg 접근 권한만으로 완성형 뮤직비디오를 자율 제작했습니다. [출처](https://news.hada.io/topic?id=31522)

네 번의 시도 모두 완성 영상이 나왔고, 세 번은 비슷한 창작 품질을 보였습니다.

이 실험이 주목받는 이유는 성능 비교보다 구조에 있습니다. 리서치-생성-편집을 단일 에이전트가 순서대로 처리하는 파이프라인이 실제로 작동한다는 점을 보여줍니다. 도구 접근 범위와 예산 상한을 명확히 설정하면, 멀티스텝(Multi-step) 미디어 제작 자동화가 현재 수준에서도 실용적임을 확인할 수 있습니다.

오늘 신호들이 공통으로 가리키는 방향은 하나입니다. AI 도구의 화제는 "무엇을 할 수 있는가"에서 "얼마나 측정 가능하고 안전하게 쓸 수 있는가"로 무게 중심이 이동하고 있습니다.
