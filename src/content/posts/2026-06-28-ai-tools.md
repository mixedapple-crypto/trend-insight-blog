## 1. K8s 에이전트 실전 비교: 모델별 행동 패턴·비용 대비 성능

**Claude Code**, **Gemini CLI**, Codex CLI를 동일한 클러스터 장애 10개에 투입해 9가지 모델 조합으로 비교한 실험 결과가 공개되었습니다. [출처](https://yozm.wishket.com/magazine/detail/3817)

핵심 발견은 "비싼 모델 = 더 나은 운영"이 아니라는 점입니다. **Claude Sonnet 4.6**이 플래그십 대비 효율에서 앞섰습니다.

모델별 행동 패턴 차이도 뚜렷합니다.

- Claude: 신중 우선, 변경 전 확인 단계 삽입
- Gemini: 속도 우선, 즉각 명령 실행
- Codex: 추론 중심, 단계별 로그 출력

**실전 팁** 세 가지입니다.

첫째, 프로덕션 투입 전 실제 인시던트(incident) 플레이북으로 파일럿 테스트를 먼저 진행합니다. 둘째, `kubectl` 명령 실행 권한은 읽기 전용(read-only)으로 시작해 범위를 점진적으로 넓힙니다. 셋째, 장애 유형별로 모델을 바꿔 테스트하면 팀 환경에 맞는 조합을 찾을 수 있습니다.

강의 시에는 "같은 프롬프트, 다른 모델"로 결과를 비교하는 실습을 구성하면 모델 선택 기준을 직관적으로 전달할 수 있습니다.

## 2. 프롬프트 인젝션 방어 실험: 2,000명의 해킹 시도를 막은 방법

`hackmyclaw.com`이라는 공개 보안 실험에서 2,000명이 6,000회 이상 시도해 AI 어시스턴트 Fiu의 `secrets.env` 파일을 탈취하려 했습니다. [출처](https://news.hada.io/topic?id=30881)

방어에 성공한 수단은 소수 줄의 시스템 프롬프트 규칙이었습니다. 복잡한 인프라 보안이 아니라 명시적 거부 지시가 핵심이었습니다.

**실전 적용 체크리스트**는 다음과 같습니다.

```
1. 시스템 프롬프트에 민감 데이터 접근 거부 규칙을 명시합니다.
2. 환경 변수는 모델이 읽을 수 있는 컨텍스트에서 분리합니다.
3. 배포 전 프롬프트 인젝션(prompt injection) 시나리오로 자체 검증합니다.
```

AI 어시스턴트를 프로덕션에 배포할 때 이 패턴은 기본 보안 레이어로 삽입해야 합니다. 강의에서는 "공격 시나리오를 직접 작성해 자신의 시스템을 테스트"하는 실습으로 활용할 수 있습니다.

## 3. Claude Tag: 슬랙 상주형 AI 팀원 사용법

Anthropic이 **Claude Tag**를 공개했습니다. 슬랙(Slack) 채널에 팀원처럼 상주하며 코드 리뷰·문서 작성·프로젝트 조율을 처리합니다. [출처](https://yozm.wishket.com/magazine/detail/3821)

사용 흐름은 단순합니다. 채널에 초대 후 멘션만으로 작동합니다. 별도 툴 전환 없이 슬랙 대화 내에서 컨텍스트(context)가 유지됩니다.

주목할 수치가 있습니다. Anthropic은 내부 코드베이스 작업의 약 **65%**를 Claude Tag로 처리한다고 밝혔습니다.

**활용 시나리오** 두 가지입니다.

- 코드 리뷰 채널에 초대해 PR(pull request) 요약을 자동 요청합니다.
- 회의 채널에서 결정 사항을 멘션으로 정리 요청합니다.

기존 AI 코딩 도구가 에디터 중심이었다면, Claude Tag는 커뮤니케이션 레이어로 이동한 구조입니다. 분산 팀 강의 환경에서도 슬랙 기반 협업 실습 시나리오로 바로 적용할 수 있습니다.

## 4. 루프 엔지니어링: 어느 계층이 지금 내 문제에 맞는가

루프 엔지니어링(loop engineering)은 프롬프트 엔지니어링의 새 계층입니다. 기존 지식을 대체하지 않고, 그 위에 쌓입니다. [출처](https://yozm.wishket.com/magazine/detail/3816)

계층 구조를 정리하면 다음과 같습니다.

| 계층 | 대상 문제 |
|---|---|
| 프롬프트 엔지니어링 | 단순 단일 질의 |
| 컨텍스트 엔지니어링 | 복잡한 단일 작업 |
| 하네스(harness) 엔지니어링 | 모델 통합·호출 구조 |
| 루프 엔지니어링 | 반복 피드백이 필요한 프로덕션 시스템 |

**실전 판단 기준**은 이렇습니다. 결과를 한 번 받고 끝나면 프롬프트 계층으로 충분합니다. 모델이 자체 출력을 다음 입력으로 반복 처리해야 하면 루프 계층이 필요합니다.

강사 입장에서는 "지금 수강생의 문제가 어느 계층인가"를 먼저 진단하는 프레임으로 이 구조를 커리큘럼에 삽입할 수 있습니다.

## 5. NLnet Labs 정책·GPT-5.6 제한 공개: 생태계의 두 대비

한쪽에서는 문이 닫히고, 다른 쪽에서는 문이 열립니다.

**NLnet Labs**는 프로젝트 기여물에 LLM 생성 코드·문서를 전면 금지했습니다. [출처](https://news.hada.io/topic?id=30898) 위반 시 사전 통보 없이 기여물을 삭제합니다. 오픈소스 인프라에서 보안과 유지보수성을 이유로 내린 결정입니다.

반대 방향에서 **OpenAI GPT-5.6**(Sol·Terra·Luna 코드명)이 신뢰 파트너에만 제한 공개되었습니다. [출처](https://www.latent.space/p/ainews-openai-gpt-56-sol-terra-luna) 공개 정보가 적어 구체적 성능 수치는 아직 확인되지 않습니다.

이 두 신호가 오늘 AI 도구 흐름을 요약합니다. 도구는 더 빠르게 발전하지만, 어디에 쓰고 어디에 쓰지 않을지에 대한 명시적 선택이 실무 기준으로 자리 잡고 있습니다. macOS에서 에이전트 작업 중 절전을 막아주는 오픈소스 앱 Adrafinil처럼 [출처](https://github.com/kageroumado/adrafinil), 작지만 실용적인 도구들이 이 흐름을 조용히 뒷받침하고 있습니다.
