## 1. Claude Tag: 슬랙에서 에이전트를 팀원처럼 쓰는 법

**Anthropic**이 2026년 6월 24일 **Claude Tag**를 공개했습니다.

슬랙 채널에서 `@Claude`를 멘션하면 Claude가 대화에 참여합니다. 연결된 코드베이스·데이터·도구에 직접 접근하므로, 맥락을 별도로 붙여 넣을 필요가 없습니다.

실전 사용법은 세 가지 패턴으로 정리됩니다.

첫째, PR 리뷰 요청입니다. 채널에 PR 링크를 올리고 `@Claude 이 PR의 사이드이펙트 짚어줘`처럼 멘션하면 코드베이스 맥락을 스스로 수집해 답합니다.

둘째, 문서 생성입니다. 스프린트 채널에서 `@Claude 이번 스프린트 완료 항목 요약해서 릴리스 노트 초안 작성해줘`로 요청하면 연결된 이슈 트래커 데이터를 참조합니다.

셋째, 디버깅 협업입니다. 에러 로그를 스레드에 붙이고 멘션하면, Claude가 연결된 코드베이스에서 원인 후보를 좁혀 답합니다.

팀 단위 강의 환경에서는 학습 채널에 Claude를 참여시켜 수강생 질문에 1차 답변을 맡기는 구성도 가능합니다. [출처](https://news.hada.io/topic?id=30809)

## 2. Cafe24 LLM Router: 모델 교체 없이 공급사 바꾸기

**Cafe24**가 공개한 **LLM Router**는 OpenAI 호환 API 단일 엔드포인트로 Claude·Gemini·Qwen·Llama·DeepSeek 등 100개 이상 모델에 연결합니다.

기존 방식은 공급사마다 API 스펙, 재시도 로직, 스트리밍 포맷이 달라 멀티 모델 환경을 유지하는 비용이 컸습니다.

LLM Router를 쓰면 코드 변경 없이 모델 이름만 교체해 공급사를 바꿀 수 있습니다.

```python
# 기존 OpenAI 코드 그대로, model 값만 바꿔 Claude로 전환
client = OpenAI(base_url="https://llm-router.cafe24.com/v1", api_key="...")
response = client.chat.completions.create(
    model="claude-opus-4-8",  # 또는 "gemini-2.5-pro" 등
    messages=[{"role": "user", "content": "안녕"}]
)
```

실전 팁으로, 비용이 높은 작업은 Claude Opus로, 반복성 높은 분류 작업은 Llama 계열로 라우팅하는 **모델 계층화 전략**을 단일 코드베이스에서 구현할 수 있습니다.

강의에서는 OpenAI SDK 코드를 그대로 두고 모델 파라미터만 바꿔 여러 공급사를 비교하는 실습에 바로 활용할 수 있습니다. [출처](https://news.hada.io/topic?id=30813)

## 3. Simon Willison의 SQLite 자동화: Claude Code로 데이터 배포 파이프라인 구성

Simon Willison이 Mozilla의 `mdn/browser-compat-data`(JSON 형식)를 약 66MB SQLite 데이터베이스로 변환하는 작업을 **Claude Code Opus 4.8**과 **sqlite-utils**로 수행했습니다. [출처](https://simonwillison.net/2026/Jun/24/browser-compat-db)

핵심은 변환 이후 배포 방식입니다. GitHub Actions 워크플로가 컴파일된 데이터베이스를 고아 브랜치(orphan branch)에 강제 푸시(force-push)하면, CORS가 허용된 URL로 외부에서 바로 접근할 수 있습니다.

이 패턴을 재활용하면 정기적으로 갱신되는 데이터셋을 별도 서버 없이 GitHub에서 쿼리 가능한 형태로 배포할 수 있습니다.

실전 적용 단계는 세 단계입니다.

1. `sqlite-utils insert` 또는 Claude Code에 변환 스크립트 생성 요청
2. GitHub Actions에서 스케줄 트리거로 자동 변환·커밋
3. 고아 브랜치 URL을 Datasette Lite에 연결해 브라우저에서 즉시 쿼리

브라우저 호환성 데이터 외에도 공공 데이터 API 스냅샷, 패키지 버전 이력 등 주기적으로 갱신되는 정형 데이터에 동일하게 적용할 수 있습니다.

## 4. OpenAI Jalapeño: 추론 칩이 개발자에게 의미하는 것

**OpenAI**와 **Broadcom**이 LLM 추론(inference) 전용 칩 **Jalapeño**를 공개했습니다. 설계 착수부터 테이프아웃(tape-out)까지 9개월이 걸렸으며, 범용 GPU 대신 LLM 연산에 특화된 구조를 채택했습니다. [출처](https://news.hada.io/topic?id=30806)

API를 호출하는 개발자 입장에서 직접 칩을 다룰 일은 없습니다. 그러나 OpenAI 인프라에 이 칩이 투입되면 API 응답 지연과 토큰당 비용이 줄어드는 방향으로 영향이 전달됩니다.

실시간 채팅, 자동완성, 스트리밍 출력처럼 지연에 민감한 서비스를 OpenAI API로 구성한 팀이라면 코드 변경 없이 성능 개선 효과를 기대할 수 있습니다.

## 5. 에이전트 스킬 보안 맹점: 정적 스캔의 한계와 대응

보안 연구팀이 악성 에이전트(Agent) 스킬이 Cisco·NVIDIA·skills.sh의 정적 스캐너를 모두 통과해 인스타그램을 통해 2만 6천 명 이상에게 배포된 사례를 공개했습니다. [출처](https://www.itworld.co.kr/article/4189141/%ec%95%88%ec%a0%84-%ed%8c%90%ec%a0%95-%eb%b0%9b%ec%9d%80-ai-%ec%97%90%ec%9d%b4%ec%a0%84%ed%8a%b8-%ec%8a%a4%ed%82%ac%ec%9d%98-%eb%b0%b0%ec%8b%a0%ec%a0%95%ec%a0%81-%ec%8a%a4)

공격 구조는 단순합니다. 악성 페이로드를 제출 파일 안에 넣지 않고 외부 URL 리다이렉트 뒤에 숨겼습니다. 스캐너는 제출 시점의 파일만 검사하므로 런타임 동작을 탐지하지 못했습니다.

실무 대응 체크리스트는 세 가지입니다.

- 에이전트 스킬이 호출하는 외부 도메인 목록을 화이트리스트로 관리합니다.
- 런타임 네트워크 트래픽을 모니터링해 예상 외 도메인 접근을 알림으로 처리합니다.
- 서드파티 스킬은 npm 패키지처럼 소프트웨어 공급망(software supply chain) 보안 프로세스에 포함시킵니다.

오늘 신호들을 한 줄로 묶으면, AI 도구는 슬랙·칩·라우터·SQLite 파이프라인으로 더 깊이 인프라에 내려앉는 중입니다. 그 만큼 보안과 비용 가시성 관리가 기능 도입과 동시에 따라붙어야 합니다.
