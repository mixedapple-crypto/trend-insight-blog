## 1. Claude Code로 535,496줄 Zig→Rust 마이그레이션 자동화

**Bun** 런타임 팀이 전체 코드베이스를 Zig에서 Rust로 다시 작성 중입니다.

직접 계기는 안정성 문제입니다. 가비지 컬렉션 기반 JavaScript 엔진과 수동 메모리 관리 언어를 혼용하면서 예측하기 어려운 충돌이 누적됐습니다.

팀이 선택한 방법은 사람이 1년 이상 수동 번역하는 대신, **Claude Code**의 동적 워크플로(dynamic workflow)를 활용해 **535,496줄** 전체를 자동 이전하는 것입니다.

Claude Code는 코드 구조를 파악하고 언어 간 매핑 규칙을 학습한 뒤 체계적으로 변환을 처리합니다.

인간 번역자가 투입되면 통일성 없는 스타일, 누락된 엣지 케이스가 생기지만, 모델 기반 전환은 일관성을 유지합니다.

**실전 팁:** 대규모 언어 간 마이그레이션에 Claude Code를 적용할 때는 변환 범위를 모듈 단위로 나눠 검증 루프를 짧게 유지하는 것이 안전합니다. 전체를 한 번에 넘기기보다 한 디렉터리씩 변환→테스트→수정 순서를 반복하면 오류 추적이 수월합니다.

레거시 코드베이스 마이그레이션이나 언어 전환을 고려하는 팀에게 현실적인 기준점을 제시하는 사례입니다. [출처](https://news.hada.io/topic?id=31263)

## 2. GitHub Copilot: 도구를 개선했더니 성능이 나빠진 구조적 이유

GitHub은 **Copilot** 코드 리뷰 기능을 Unix 방식의 코드 탐색 도구로 전환한 경험을 공개했습니다.

결과는 역설적이었습니다. 도구 자체는 강력해졌지만, 초기 리뷰 품질은 오히려 떨어졌습니다.

원인은 에이전트(Agent) 워크플로를 그대로 둔 채 도구만 교체했기 때문입니다. 에이전트가 풀 리퀘스트(PR) 관련 증거를 수집하는 순서와 방식이 새 도구 구조와 맞지 않았습니다.

팀은 워크플로를 PR 증거 수집에 최적화하도록 재설계했고, 이후 리뷰 비용이 낮아지고 품질이 회복됐습니다.

**실전 팁:** Copilot 코드 리뷰를 쓸 때 기본 설정의 광범위한 스캔보다 변경된 diff 범위에 집중하도록 설정을 조정하면 불필요한 노이즈를 줄일 수 있습니다. 도구를 바꿀 때는 반드시 에이전트 워크플로 전체를 함께 검토해야 합니다.

강의에서 "도구 업그레이드 = 성능 향상"이라는 공식이 얼마나 위험한 가정인지 보여주는 실증 사례입니다. [출처](https://github.blog/ai-and-ml/github-copilot/better-tools-made-copilot-code-review-worse-heres-how-we-actually-improved-it)

## 3. ChatGPT: 미국 소상공인 400만 명의 첫 번째 실무 도구

**OpenAI**에 따르면 미국 소상공인 약 400만 명이 **ChatGPT**를 핵심 업무에 활용 중입니다.

주요 용도는 마케팅 카피 작성, 고객 문의 응대, 재무 계획 정리입니다.

이들은 ChatGPT를 "첫 번째 고용"으로 표현합니다. 직원 없이 창업 초기를 버티는 수단으로 쓴다는 뜻입니다.

**실전 팁:** 소상공인 대상 강의나 B2B SaaS를 개발한다면 ChatGPT 연동을 기본값으로 설계하는 것이 현실적입니다. 사용자들이 이미 ChatGPT 워크플로에 익숙하기 때문에, 별도 학습 없이 진입할 수 있는 인터페이스가 채택률을 높입니다.

창업 초기 비용을 낮추는 도구로 자리 잡으면서, AI 통합이 B2B 제품의 기본 요건으로 바뀌는 흐름을 보여줍니다. [출처](https://openai.com/index/ai-first-hire-small-business)

## 4. 도구 신호 종합 — 실속과 과장 사이

오늘 남은 신호 중 주목할 것은 **llm 0.31.1** 버그 수정입니다.

일부 AI 모델이 인자 없이 함수를 호출할 때 빈 JSON 오류가 발생하던 문제를 수정했습니다. 함수 호출(function calling) 워크플로를 쓰는 환경이라면 업데이트가 권장됩니다. [출처](https://simonwillison.net/2026/Jul/9/llm)

오늘 흐름을 묶으면 하나의 대비가 선명합니다. Claude Code의 50만 줄 자동화처럼 AI 도구가 실제 생산 공정에 깊숙이 들어온 사례가 있는 반면, Copilot 사례는 도구 성능만 믿고 워크플로를 방치하면 결과가 나빠진다는 것을 보여줍니다. 도구를 바꿀 때 설계를 함께 바꿔야 한다는 원칙이, 오늘 신호 전체를 관통합니다.
