## 1. sqlite-utils 4.1: 인라인 코드로 데이터 삽입, 타입 오버라이드, 인덱스 관리

**sqlite-utils 4.1**이 2026년 7월 11일 공개됐습니다.

가장 실용적인 추가 기능은 `insert`와 `upsert` 명령의 `--code` 옵션입니다.

기존에는 별도 Python 파일을 작성하거나 파이프로 데이터를 넘겨야 했습니다. 이제는 CLI 한 줄 안에 `rows()` 함수를 정의해 바로 실행할 수 있습니다.

```bash
sqlite-utils insert data.db creatures \
  --code 'def rows():
    yield {"id": 1, "name": "Cleo"}'
```

스크립트 파일 없이 테스트 데이터를 빠르게 넣거나, 강의 시연에서 의존성 없이 예제를 실행할 때 유용합니다.

두 번째 변경은 **`--type` 옵션**입니다. CSV 가져오기(import) 시 자동 추론된 컬럼 타입을 덮어쓸 수 있습니다.

```bash
sqlite-utils insert data.db zips zips.csv \
  --csv --type zip_code TEXT
```

ZIP 코드처럼 앞자리 0이 숫자로 잘리는 문제를 막습니다. 날짜·주민번호·코드값 등 "숫자처럼 생긴 문자열" 컬럼을 다룰 때 즉시 적용 가능합니다.

세 번째는 `drop_index()` 메서드와 `drop-index` 명령입니다. 인덱스 이름으로 삭제하며, `--ignore` 플래그로 존재하지 않는 인덱스에 대한 오류를 무시할 수 있습니다.

```bash
sqlite-utils drop-index data.db idx_name --ignore
```

자동화 스크립트에서 멱등성(idempotency)을 확보할 때 유용합니다. [출처](https://simonwillison.net/2026/Jul/11/sqlite-utils)

## 2. 모델 실측 비교: 복잡도별 신뢰도가 갈리는 구조

12개 AI 모델을 동일한 앱 4종으로 실측한 벤치마크가 공개됐습니다.

테스트 대상 앱은 레이트레이서(raytracer) 미로, 3D 루빅스 큐브, 계산기, 콘웨이(Conway) 생명 게임 4가지입니다. 각 모델·과제 조합은 5회씩 반복 실행했고, 성공률·비용·소요 시간을 기록했습니다.

복잡한 과제에서는 **GPT-5.6 Sol**과 **Claude Fable 5**가 앞섰습니다. 단순 과제에서는 여러 모델이 고르게 성공률을 보였습니다.

실전 팁은 두 가지입니다.

첫째, 마케팅 벤치마크보다 과제 복잡도별 성공률을 기준으로 모델을 선택합니다. 복잡한 UI 컴포넌트나 알고리즘 구현 과제에는 상위 모델을, 반복적 단순 코드 생성에는 비용 대비 효율을 따집니다.

둘째, 동일 과제를 5회 반복하는 방식은 개인 워크플로에도 적용할 수 있습니다. 모델 하나를 선택하기 전 같은 프롬프트를 3회 이상 실행해 일관성을 확인합니다. [출처](https://news.hada.io/topic?id=31342)

## 3. 오늘의 구조: 조용한 날의 실속 신호

오늘은 대형 모델 발표나 기능 공개 없이, 실무 도구의 세밀한 개선이 중심이었습니다.

sqlite-utils 4.1의 `--code` 옵션은 "파일 없이 CLI만으로 완결되는 워크플로"라는 방향을 한 걸음 더 밀고 나간 사례입니다. 모델 비교 벤치마크는 "어느 모델이 강한가"보다 "어느 복잡도에서 신뢰할 수 있는가"라는 질문을 던집니다.

화려한 발표가 없는 날일수록, 이미 쓰고 있는 도구의 새 옵션 하나가 실제 시간을 아끼는 경우가 많습니다.
