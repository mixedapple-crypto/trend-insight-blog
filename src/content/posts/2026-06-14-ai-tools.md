## 1. Pyodide + PyPI: WASM 휠 직접 배포로 브라우저 Python 생태계 개방

**Pyodide 314.0**부터 패키지 관리자가 C/Rust 확장을 포함한 WASM 휠을 PyPI에 직접 올릴 수 있습니다.
기존에는 Pyodide 코어 팀이 수동으로 빌드·호스팅해야 했고, 300여 개 패키지만 쓸 수 있었습니다.

**어떻게 쓰는가:**
1. 패키지를 WASM 타깃으로 빌드합니다.
2. 플랫폼 태그를 `pyemscripten_*_wasm32`로 지정한 뒤 일반 휠과 동일하게 PyPI에 업로드합니다.
3. Pyodide 환경에서 `micropip.install("패키지명")`으로 바로 설치합니다.

실전 팁은 두 가지입니다.

첫째, C 확장이 없는 순수 Python 패키지는 별도 작업 없이 기존처럼 `micropip`으로 설치되므로, WASM 빌드 작업은 C/Rust 의존성이 있는 패키지에만 집중하면 됩니다.

둘째, 이번 변경의 기반은 **PEP 783**입니다. 패키지 빌드 설정을 바꾸기 전에 해당 PEP의 플랫폼 태그 규칙을 먼저 확인하십시오. [출처](https://simonwillison.net/2026/Jun/13/publishing-wasm-wheels)

같은 날 공개된 **luau-wasm 0.1a0**도 이 흐름 위에 있습니다. Roblox의 Lua 방언인 Luau를 PyPI에서 설치해 브라우저 내 샌드박스 환경에서 바로 실행할 수 있습니다. 웹 기반 IDE에서 Luau를 가르치는 강사라면 별도 런타임 없이 바로 활용 가능합니다. [출처](https://simonwillison.net/2026/Jun/13/luau-wasm)

## 2. MCP 서버 10종: 자연어로 PostgreSQL·BigQuery 연결하는 실전 설정법

모델 컨텍스트 프로토콜(MCP, Model Context Protocol) 서버를 이용하면 클로드 코드(Claude Code)·커서(Cursor)·제미나이(Gemini) CLI에서 SQL 없이 데이터베이스를 다룰 수 있습니다.

PostgreSQL, Amazon Aurora(MySQL), Google BigQuery 등 주요 플랫폼용 공식 MCP 서버가 정리되었습니다.

**설정 방법:**
MCP 설정 파일에 JSON 블록을 추가하면 됩니다.
BigQuery 예시는 아래와 같습니다.

```json
{
  "mcpServers": {
    "bigquery": {
      "command": "npx",
      "args": ["-y", "@google-cloud/mcp-server-bigquery"]
    }
  }
}
```

이후 클로드 코드에서 `PROJECT_ID의 데이터셋 목록을 보여줘`처럼 자연어로 요청하면 스키마 메타데이터와 쿼리 결과를 반환합니다.

실전 팁은 세 가지입니다.

첫째, AWS·GCP 공식 서버는 관리형 호스팅을 제공하므로 보안 설정 부담이 줄어듭니다.

둘째, 자연어 쿼리로 스키마를 빠르게 탐색한 뒤 실제 SQL은 에이전트(Agent)가 생성하게 맡기면 디버깅 속도가 올라갑니다.

셋째, 데이터베이스 접근 권한은 읽기 전용 계정으로 제한하는 것이 베스트 프랙티스입니다. [출처](https://www.itworld.co.kr/article/4184249/)

## 3. macOS 로컬 코딩 에이전트: llama.cpp + Gemma로 오프라인 추론 구성

**llama.cpp**와 **Gemma 4 26B-A4B GGUF** 모델을 조합하면, 애플 실리콘 맥에서 OpenAI 호환 API를 오프라인으로 구동할 수 있습니다.

M1 Max + macOS 15.7.7 환경에서 Metal 가속으로 테스트가 완료되었으며, 텍스트와 이미지 입력을 모두 처리합니다.

**구성 단계:**
1. llama.cpp를 빌드합니다(`METAL=1 make`).
2. Gemma 4 26B-A4B GGUF 모델 파일을 다운로드합니다.
3. `llama-server` 명령으로 로컬 OpenAI 호환 엔드포인트를 실행합니다.
4. 기존 클라이언트의 `base_url`을 `http://localhost:8080`으로 변경합니다.

실전 팁으로는, 민감한 고객 코드베이스를 다루거나 API 속도 제한(Rate Limit)이 문제가 되는 환경에서 특히 유용합니다.

클라우드 추론 대비 레이턴시(Latency)는 높을 수 있으나, 네트워크 단절 상황에서 워크플로를 유지하는 데 실질적인 가치가 있습니다. [출처](https://news.hada.io/topic?id=30461)

## 4. 구글 기본 설정 변경: 내 파일·음성이 AI 학습 데이터로 들어가기 전에

구글이 **Search Services History** 설정을 새로 도입하면서, 구글 렌즈·검색·제미나이 라이브(Gemini Live)에서 사용한 이미지·파일·영상·음성이 기본값으로 AI 학습 데이터에 포함될 예정입니다.

**지금 바로 해야 할 일:**
1. 구글 계정 설정으로 이동합니다.
2. `활동 관리(Activity Management)`를 선택합니다.
3. `Search Services History` 항목에서 `미디어 저장(Save Media)`을 끕니다.
4. 해당 메뉴가 아직 표시되지 않는다면, `웹 및 앱 활동(Web & App Activity)` 전체를 꺼두십시오.

클라이언트 자료나 내부 코드 스크린샷을 자주 검색에 활용하는 개발자·강사라면 지금 바로 확인이 필요합니다.

기본값이 '수집 허용'으로 설정된 구조이므로, 명시적으로 끄지 않으면 이후 학습 데이터 포함 여부를 소급해 막기 어렵습니다. [출처](https://www.itworld.co.kr/article/4184244/)

## 5. DevOps AI 도구 실측치 vs. 애플 AI 실행력 논란: 수치와 현실 사이

가트너(Gartner) 분석가 재키 스완슨(Jackie Swanson)에 따르면, AI 지원 코딩과 자동화 테스트 생성을 도입한 DevOps 팀의 사이클 타임(Cycle Time)은 **20~40%** 단축되었습니다.

깃허브 코파일럿(GitHub Copilot)·아마존 Q 디벨로퍼(Amazon Q Developer) 같은 도구가 보일러플레이트·단위 테스트·인프라 코드를 처리하고, AIOps 플랫폼이 이상 감지·알림 연관·수정 제안을 맡는 구조입니다.

핵심은 '코딩 보조' 하나가 아니라, 코딩·CI/CD·로깅·인시던트 대응·보안 전 구간에 AI를 레이어로 깔아야 수치가 나온다는 점입니다. [출처](https://www.itworld.co.kr/article/4183887/)

반면 **WWDC26**에서 AI를 전면에 내세운 애플의 행보는 대조적인 신호를 보냅니다.

발표 직후 일부 기능이 전 기기에서 작동하지 않는 것이 확인되었고, 모건 스탠리·UBS·바클레이스 등 분석가들은 점진적 출시와 기존 솔루션과의 기능 격차가 아이폰 교체 수요를 단기 견인하기 어렵다고 평가합니다.

수치 20~40%는 전 구간 통합 사례에서 나온 값이고, 애플처럼 데모와 실제 배포 사이 간극이 크면 그 수치는 의미를 잃습니다.

오늘 도구 흐름을 한 줄로 정리하면: 생태계 개방(Pyodide·MCP)과 로컬 에이전트 구성이 실무자의 선택지를 넓히는 방향으로 수렴하고 있으며, 수치를 낼 수 있는 것은 전 구간 통합이지 단일 도구 교체가 아닙니다. [출처](https://www.itworld.co.kr/article/4183866/)
